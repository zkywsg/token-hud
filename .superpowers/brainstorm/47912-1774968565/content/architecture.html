整体架构

两个独立项目，通过一个 JSON 文件解耦

项目 1 · 开源库

token-state

TypeScript · npm 发布

TokenTracker — 核心类，管理所有 service 的配额状态
consume() — 手动上报用量（tokens / time / money）
Session — 追踪单次会话的消耗量
wrapOpenAI / wrapAnthropic — 可选 SDK 插件，自动拦截
状态持久化 → 写入 state.json
→
📄
~/.token-hud/
state.json
共享状态文件
→
FSEvents
监听变化
项目 2 · macOS App

token_hud

Swift + SwiftUI · 本地 App

StateWatcher — FSEvents 监听 state.json，触发 UI 更新
NotchOverlay — 在刘海两侧渲染浮层窗口
左侧 — 各 service 配额剩余量
右侧 — 当前 session 消耗量
设置面板 — 配置 state.json 路径、显示偏好
解耦关键： 两个项目只通过 state.json 通信。token-state 库不知道 HUD 的存在，HUD 不知道数据怎么产生——任何人都可以写入这个文件，任何 UI 都可以读取它。
