state.json 数据结构

两个项目之间的唯一契约 — 设计好了后两边都依赖它

示例文件

{
  "version": 1,
  "updatedAt": "2026-03-31T10:00:00Z",
  "services": {
    "claude": {
      "label": "Claude Max",
      "quotas": [
        {
          "type": "time",
          "total": 18000,
          "used": 3600,
          "unit": "seconds",
          "resetsAt": "2026-04-30T..."
        },
        {
          "type": "tokens",
          "total": 1000000,
          "used": 150000,
          "unit": "tokens"
        }
      ],
      "currentSession": {
        "id": "sess_abc123",
        "startedAt": "2026-03-31T09:50:00Z",
        "tokens": 1500,
        "time": 142.5
      }
    },
    "openai": { ... }
  }
}

字段说明

version
schema 版本号，方便未来升级时做兼容处理
quotas[].type
配额类型：tokens · time · money · requests
quotas[].total / used
总配额与已用量，剩余 = total − used，HUD 直接计算
quotas[].resetsAt
可选，配额重置时间（如月度订阅重置日）
currentSession
当前活跃会话，null 表示无会话。HUD 右侧显示这里的数据。
⚡ 原子写入
TS 库写文件时先写 state.tmp.json，再原子 rename，避免 HUD 读到半写状态
