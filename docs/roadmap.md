# 路线图

这个文件记录更长期的方向。当前逐步执行的工作保留在 `PLAN.md`。

## 产品方向

- 让 AI 用量一眼可见，避免用户反复打开各个平台 dashboard。
- 保持 HUD 轻量、本地优先，并安全处理敏感凭据。
- 同时支持面向刘海区域的展示，以及适合非刘海设备或特殊工作流的可移动浮动面板。

## 近期优先级

- 确认最终 HUD 视觉设计，并替换 README 中的截图占位。
- 收紧每个平台的空数据和错误状态展示。
- 让 session-key、API-key、本地认证和控制台 cookie 类型的平台配置流程保持一致。
- 保持 widget 预设与各平台实际可提供的指标一致。

## 长期优先级

- 在本仓库记录 `state.json` schema，让外部写入方不必只依赖上游 `token-state` README。
- 在能提升可测试性的地方，把特定 provider 的拉取/解析逻辑与 UI 关注点拆开。
- 添加新 provider 或额度类型前，扩展解析器和格式化测试。
- 决定刘海 HUD 和浮动面板应继续作为同一套 overlay 系统，还是拆成两个展示模式。
- 补充发布文档，包括签名、权限、开机启动行为和首次启动配置。
